# Airygen SEO

Модульный [плагин SEO для WordPress](https://www.airygen.com/ru), объединяющий управление SEO на странице, структурированные данные, технический SEO, рабочие процессы внутренних ссылок и вывод данных для ИИ — при этом весь процесс редактирования остаётся в WordPress.

## Системные требования

- WordPress 6.3+
- PHP 8.1+

## Возможности

### SEO на странице
- Пользовательский заголовок, описание, канонический URL и директивы robots для каждой записи/страницы
- Проверка ширины в пикселях и калькулятор SEO-оценки
- Анализ фокусной ключевой фразы

### Структурированные данные
- JSON-LD Schema для статей, сайта, организации, авторов и хлебных крошек
- Schema продуктов WooCommerce

### Социальные сети & Meta
- Метаданные Open Graph (Facebook) и карточки Twitter/X
- Подписи RSS и верификация сайта (Google, Bing, Yandex)

### Технический SEO
- Генерация и управление XML-картой сайта
- Управление robots.txt и мета-директивами robots
- Управление редиректами (301 / 302 / 410) и журналирование ошибок 404
- Мониторинг битых ссылок и мгновенная индексация (Google Indexing API)
- Альтернативные языковые ссылки Hreflang

### Контент & UX
- Хлебные крошки (HTML + Schema)
- Блок оглавления
- Блок похожих записей
- SEO изображений — автоматическое добавление `alt`/`title` во время выполнения с настраиваемыми шаблонами
- Менеджер фрагментов кода (внедрение в head / body / footer)

### Внутренние ссылки
- Подсчёт и отслеживание распределения ссылок
- Предложения внутренних ссылок
- Визуализация тематических кластеров

### Расширения
- Локальный SEO, SEO автора, SEO таксономий
- SEO для WooCommerce
- Интеграции уведомлений

### Инструменты ИИ
- `llms.txt` — индекс сайта, доступный для чтения агентами ИИ
- Markdown for Agents — экспорт контента в формате Markdown для LLM

## Разработка

### Предварительные требования

- Docker
- pnpm 9+

### Запуск среды разработки

```bash
make dev
```

Запускает контейнеры Docker, устанавливает WordPress, активирует плагин и запускает серверы разработки frontend. Откройте `http://localhost:9000` (admin: `admin` / `admin`).

> **Чистая база данных:** По умолчанию контейнер MySQL импортирует данные из `.docker/wp/schema/backup.sql`. Для чистой установки WordPress закомментируйте эту строку в `docker-compose.yml` перед запуском `make up`:
> ```yaml
> # - ./.docker/wp/schema/backup.sql:/docker-entrypoint-initdb.d/backup.sql:ro
> ```

### Основные команды

| Команда | Описание |
|---|---|
| `make up` / `make down` | Запустить / остановить контейнеры Docker |
| `make wp.init-dev-site` | Установить WordPress + активировать плагин + WooCommerce |
| `make dev.admin` | Сервер разработки SPA для администратора |
| `make dev.block-editor` | Сервер разработки редактора блоков |
| `make dev.classic-editor` | Сервер разработки классического редактора |
| `make tests` | Запустить интеграционные тесты PHPUnit |
| `make phpcs` | PHP CodeSniffer |
| `make phpstan` | Статический анализ PHP |
| `make lint` | ESLint |
| `make lint.types` | Проверка типов TypeScript |
| `make i18n.check` | Перегенерировать POT и синхронизировать файлы PO |
| `make i18n.build` | Сгенерировать файлы переводов MO и JSON |

### Технологический стек

- **PHP** 8.1+ — автозагрузка PSR-4, пространство имён `Airygen\`, модульная архитектура с приоритетом функций
- **React + TypeScript** 18 / 5 — три точки входа: SPA администратора, редактор блоков, классический редактор
- **Сборка** — `@wordpress/scripts` (webpack), pnpm workspaces
- **CSS** — Tailwind CSS 3, Sass
- **Тестирование** — PHPUnit 8, `wp-phpunit`, среда WordPress на основе Docker

## Архитектура

Исходный код находится в `src/Modules/<Feature>`, каждый модуль разделён на слои `Admin/`, `Public/`, `Runtime/` и `Domain/`. Общая инфраструктура находится в `src/Admin`, `src/Public` и `src/Support`. Полное руководство по структуре смотрите в `guidelines/structure.md`.
